# 要件定義

## 1. 目的

CRUD機能およびWebアプリケーション開発における各種技術（HTTP通信、フロント・バックエンド連携、認証・認可）の理解を目的とする。

## 2. 利用者

個人開発者を想定するが、第三者でも利用可能なシンプルなタスク管理アプリとする。

## 3. 機能要件

### 3.1 認証機能

- ユーザー登録・ログイン（メールアドレス＋パスワード）
- セッションベース認証
- ユーザー種別: 一般ユーザー(user), 管理者(admin)
- 権限管理: ロールベースアクセス制御

### 3.2 タスク管理機能

以下の機能を提供する。

- タスクの追加（Create）
- タスクの参照（Read）
- タスクの更新（Update）
- タスクの削除（Delete）
- タスクの検索（キーワード検索）
- タスクのステータスによるフィルタリング（grep：ステータスごとにタスク一覧を表示する機能）

## 4. 非機能要件

- ローカル環境で動作すること
- データはデータベースに保存すること
- HTTPS環境での運用を前提
- セッション管理の安全性を確保

## 5. 制約・前提

- 認証機能を実装（ログイン必須）
- 複数ユーザー対応を行う
- タスクはユーザー単位で管理する
- DB: PostgreSQLに移行（学習目的で無料DB利用）
- 外部認証（Google等）は対象外
