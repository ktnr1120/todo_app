# 要件定義

## 1. 目的

CRUD機能およびWebアプリケーション開発における各種技術（HTTP通信、フロント・バックエンド連携）の理解を目的とする。

## 2. 利用者

個人開発者を想定するが、第三者でも利用可能なシンプルなタスク管理アプリとする。

## 3. 機能要件

### 3.1 タスク管理機能

以下の機能を提供する。

- タスクの追加（Create）
- タスクの参照（Read）
- タスクの更新（Update）
- タスクの削除（Delete）
- タスクの検索

## 4. 非機能要件

- ローカル環境で動作すること
- データはデータベースに保存すること

## 5. 制約・前提

- 認証機能（ログイン）は実装しない
- 複数ユーザー対応は行わない
- タスクは単一ユーザーで管理する
