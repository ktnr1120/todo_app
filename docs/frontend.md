# フロントエンド設計書

## 作成日時

2026年4月13日 (ログイン機能追加: 2026年4月16日)

## 1. 目的

TODO Appのユーザーインターフェースを提供し、バックエンドAPIとの連携を通じてタスク管理機能を実現する。ログイン機能追加により、認証状態に応じたUI表示とユーザー単位のデータ管理を実現。

## 2. 利用者

個人開発者および一般ユーザー。シンプルで使いやすいUIを提供。

## 5. 機能要件

### 5.1 認証機能

- **ログイン**: フォーム入力 → POST /login → 成功時メイン画面へ遷移
- **ログアウト**: ボタンクリック → POST /logout → ログインページへ遷移
- **認証状態チェック**: ページロード時 GET /auth/status → 未認証時ログインページへリダイレクト

### 5.1 認証機能

- **ログイン**: フォーム入力 → POST /login → 成功時メイン画面へ遷移
- **ログアウト**: ボタンクリック → POST /logout → ログインページへ遷移
- **認証状態チェック**: ページロード時 GET /auth/status → 未認証時ログインページへリダイレクト

### 5.2 タスク管理機能

- **タスク作成**: フォーム入力 → POST /tasks
- **タスク一覧表示**: GET /tasks → リアルタイム更新
- **タスク更新**: ステータス変更 → PUT /tasks/:id
- **タスク削除**: 削除ボタン → DELETE /tasks/:id
- **ステータスフィルタ**: ドロップダウン選択 → GET /tasks?status=...
- **キーワード検索**: 入力 → GET /tasks?keyword=...

### 5.3 UI/UX機能

- **ローディング表示**: API呼び出し中のスピナー
- **エラーハンドリング**: ユーザー-friendlyなエラーメッセージ
- **レスポンシブ**: モバイル対応（簡易）

## 6. コンポーネント設計

### 6.1 コンポーネント構成

```
App.jsx (メインコンポーネント)
├── AuthContext (認証状態管理)
├── Login.jsx (ログインページ)
│   ├── メールアドレス入力 (input)
│   ├── パスワード入力 (input)
│   ├── ログインボタン (button)
│   └── エラーメッセージ (div.error)
├── ヘッダー (header)
│   ├── アプリタイトル (h1)
│   └── ログアウトボタン (button)
├── エラー表示 (div.error)
├── タスク作成フォーム (form.task-form)
│   ├── 入力フィールド (input)
│   └── 追加ボタン (button)
├── フィルタ・検索 (div.filters)
│   ├── ステータス選択 (select)
│   └── 検索入力 (input)
└── タスク一覧 (div.task-list)
    └── タスクカード (div.task-item) × N
        ├── タイトル (h3)
        ├── ステータス選択 (select)
        └── 削除ボタン (button)
```

### 6.2 コンポーネント仕様

#### AuthContext

- **役割**: 認証状態のグローバル管理
- **状態**:
  - user: ログイン中ユーザー情報
  - authenticated: 認証状態
- **メソッド**:
  - login(email, password): ログイン処理
  - logout(): ログアウト処理
  - checkAuth(): 認証状態確認

#### Login.jsx

- **役割**: ログインフォーム表示・処理
- **状態**:
  - email: 入力メールアドレス
  - password: 入力パスワード
  - error: エラーメッセージ
  - loading: ログイン中状態

#### App.jsx

- **役割**: メインコンテナ、全状態管理
- **状態**:
  - tasks: タスク配列
  - loading: ローディング状態
  - error: エラーメッセージ
  - newTaskTitle: 新規タスク入力
  - filterStatus: フィルタ選択
  - searchKeyword: 検索キーワード

#### 各要素の役割

- **タスク作成フォーム**: 入力バリデーション、Enterキー対応
- **フィルタ**: 即時反映（useEffect）
- **タスクカード**: 個別操作（更新/削除）

## 7. API連携仕様

### 7.1 APIクライアント（src/api/tasks.js）

- **ベースURL**: `/tasks` （proxy経由でバックエンドに転送）
- **メソッド**: GET, POST, PUT, DELETE
- **エラーハンドリング**: axiosのcatchで統一処理

### 7.2 API呼び出しパターン

- **一覧取得**: コンポーネントマウント時 + フィルタ変更時
- **作成/更新/削除**: ユーザー操作後、自動再取得
- **エラー処理**: UIにエラーメッセージ表示 + コンソールログ

### 7.3 データフロー

```
ユーザー操作 → API呼び出し → 状態更新 → UI再描画
     ↓
エラー発生 → エラーメッセージ表示
```
